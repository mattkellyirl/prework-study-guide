HTML

  • HTML stands for hyper text markup language and is the structure of webpages!
  • HTML, head, title and body tags make up the basic structure of a HTML document.
  • The head element contains information about the webpage, such as links to css/javascript files associated with the webpage.
  • The body element represents the visible content shown to the user, and contains most of the information on the webpage.
  • Text can be styled in HTML using b for bold, i for italics, em for emphasizing and u for underlining tags.
  • Text headings can be created in 6 different sizes by using the h1 to h6 tags.
  • Paragraphs can be defined on web pages using the p tag.

CSS

  • CSS stands for cascading style sheet, and is how webpages are styled!
  • CSS files are linked in the head element of the HTML document by using link and href attributes.
    • link rel="stylesheet" href="./assets/style.css"
  • CSS uses selectors to target elements on an HTML page to style them. Some examples of CSS selectors are type selectors, class selectors and id selectors.
  • Margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside of an element.
  • Commenting out CSS code is done like this: /*this code has been commented out*/

Git

  • Git is version control software!
  • git clone enter-ssh-link-from-github: This git command will clone a remote repository from GitHub to our local machine.
  • git status: This git command checks what branch we are currently on.
  • git branch: This git command will output a list of available branches.
  • git checkout -b new-branch-name: This git command creates a new branch and switches to it.
  • git add -A: This git command adds ALL new changes to the staging area.
  • git commit -m "message here": This git command commits our new changes with a message.
  • git pull origin branch-name: This git command pulls down any changes made to the remote branch on GitHub.
  • git push origin branch-name: This git command pushes the changes we made on our local repository to the remote branch on GitHub.

Javascript

  • Javascript enables functionality and interation with web applications!
  • Each line of Javascript code must end with a semi colon (;).
  • Javascript is read from top to bottom, this is called control flow! Code will execute from the first statement at the top of our .js file, down to the last statement at the bottom.
  • An array is a single variable that is used to hold a group of data. Generally, this data is related in some way. Arrays can store different types of data, such as strings (text), numbers and boolean values (true, false).
  • When storing strings inside of an array, the data must be surrounded by quotation marks. For example, circle would be written as “circle”. Numbers and boolean value data types do not need to be surrounded by quotation marks when stored in an array.
  • Each item we store in an array, is given a unique identifier called an index. In Javascript, arrays are zero indexed and sequential. This means the first item of the array will always be 0, and the following items increase in increments of 1 each time.
  • We can target an individual data type from an array by calling upon its index. For example, we will target the “triangle” data from the following array by calling upon its index number below.
    • var shapes = [“triangle”, “square”, “circle”];
    console.log(shapes[0])
  • The result from the above code will log "triangle" to the console.
  • At the most basic level, a function is a set of instructions that tells a computer how to perform a certain task.
  • A function is different from a for loop or conditional statement, because functions do not automatically execute. Instead, functions must be called by name in order to execute it.
  • A functions name is always followed by parentheses (), which includes the parameters of the function.
  • Commenting out Javascript code is done like this: /*this code has been commented out*/
